About HackerHives

You’ll work with air quality data from low-cost sensor networks used in rural and under-resourced communities. These systems matter — but they’re vulnerable to subtle cyber threats, faulty sensors, and manipulated APIs.

Real-world challenge

Detect real air quality events and evaluate sensor reliability under uncertainty.

Build practical skills

APIs, time-series workflows, anomaly detection, trust scoring, open-source tooling.

Defense mindset

Design cybersecurity recommendations to harden distributed data pipelines and IoT systems.

Why participate?

Hack like it matters. Because it does.

Schedule

Two days. Mentors. Checkpoints. Final demo.

Day 1

Kickoff + event detection

Morning
Welcome, theme overview, mentors, team formation, technical setup (API + time-series + anomaly basics)
Afternoon
Detect a real air quality event • ideation • work block • Checkpoint #1 (approach + initial reliability)
Evening
Work session • optional lightning talk (sensor attacks + integrity failures) • stand-ups • plan trust model for Day 2
Day 2

Trust model + defense + demo

Morning
Build trust score • evaluate reliability across sensors • Checkpoint #2 (logic + anomalies found)
Afternoon
Stress-test with simulated bad data • defensive recommendations • submission (repo + deck/demo) • final presentations (10–15 min)
Closing
Judging + awards • next steps • networking
